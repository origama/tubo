Minimal P2P tunnel for API services.

tubo is a small, intent-based tunnel for private swarms: relay, attach, connect, discover, and curl — without dragging a big control plane behind it.

What tubo does

Attach services

Publish an HTTP service into a private swarm with a simple name and a target URL.

Connect by name

Resolve a service and open a local listener that forwards real traffic to the right place.

Keep the network small

One binary, relay/bootstrap nodes, discovery, and a straightforward operational model.

How it flows

Relay / bootstrap

Start the relay node that anchors the swarm.

tubo relay

Attach / publish

Expose a local API or app into the swarm under a stable service name.

tubo attach http://127.0.0.1:1234 --name lmstudio

Join / configure

Point a machine at an existing swarm using a relay and a shared swarm key.

tubo join --relay /ip4/1.2.3.4/tcp/4001/p2p/12D3... --swarm-key ./swarm.key

Connect / consume

Open a local listener that reaches the published service and try a real request.

tubo connect lmstudio --local 127.0.0.1:51234
curl http://127.0.0.1:51234/v1/dummy

Why tubo

There are many tunnels. tubo stays small, composable, and explicit: it lets you wire up relay, attach, connect, and discovery without hiding the network behind magic.

Built for operators

It ships with a compact CLI, real discovery, release assets, and a GitHub-pages-friendly site.
