Flat libp2p API Tunnel — Architecture

Nessun control plane per registrare servizi/API. Discovery distribuita via libp2p pubsub/rendezvous/DHT. Solo due processi operativi: edge proxy e connector/agent.
flat-first / no service registry / p2p discovery

1) Cosa avvii

  • Edge gateway pubblico
  • Connector/agent sull’origin
  • Bootstrap/relay opzionali per la rete

2) Come avviene discovery

  • Pubsub come meccanismo primario
  • TTL + heartbeat + record firmati
  • Cache locale all’edge

3) Cosa passa sul tunnel

  • HTTP method, path, query, headers
  • Response status, body, streaming
  • SSE, chunked, payload binari
